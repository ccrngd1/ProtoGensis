# RAG Verification Comparison

A working demo comparing three approaches to RAG output verification:
1. **LLM-as-Judge** (Claude on Bedrock)
2. **NLI Claims** (DeBERTa-v3-large on SageMaker)
3. **Real-Time Encoder** (Fast & Faithful approach on SageMaker)

Built as a follow-up to CC's 2023 TDS article ["How to Measure the Success of Your RAG-based LLM System"](https://towardsdatascience.com/how-to-measure-the-success-of-your-rag-based-llm-system-874a232b27eb).

## Features

- ✅ Complete RAG pipeline (Bedrock Knowledge Base + Claude generation)
- ✅ Three parallel verification approaches with accuracy/latency/cost comparison
- ✅ Mock mode for local testing without AWS credentials
- ✅ 30 labeled test cases (faithful, partial, and full hallucinations)
- ✅ Automated report generation with markdown tables
- ✅ Production-ready SageMaker deployment scripts

## AWS Deployment

### Prerequisites

- AWS Account with Bedrock and SageMaker access
- AWS CLI configured with appropriate credentials
- Python 3.9+

### Step 1: Set Up Bedrock Knowledge Base

1. Create a Bedrock Knowledge Base:
   ```bash
   aws bedrock-agent create-knowledge-base \
     --name "rag-verification-kb" \
     --role-arn "arn:aws:iam::YOUR_ACCOUNT:role/BedrockKBRole" \
     --knowledge-base-configuration '{
       "type": "VECTOR",
       "vectorKnowledgeBaseConfiguration": {
         "embeddingModelArn": "arn:aws:bedrock:us-east-1::foundation-model/amazon.titan-embed-text-v1"
       }
     }'
   ```

2. Create a data source and upload documents:
   ```bash
   # Upload documents to S3
   aws s3 cp ./docs/ s3://your-kb-bucket/docs/ --recursive

   # Create data source
   aws bedrock-agent create-data-source \
     --knowledge-base-id <KB_ID> \
     --name "docs" \
     --data-source-configuration '{
       "type": "S3",
       "s3Configuration": {
         "bucketArn": "arn:aws:s3:::your-kb-bucket"
       }
     }'

   # Ingest documents
   aws bedrock-agent start-ingestion-job \
     --knowledge-base-id <KB_ID> \
     --data-source-id <DATA_SOURCE_ID>
   ```

3. Note your Knowledge Base ID for the `.env` file.

### Step 2: Deploy NLI Model to SageMaker

Deploy DeBERTa-v3-large NLI cross-encoder:

```python
# deploy_nli.py
from sagemaker.huggingface import HuggingFaceModel
import sagemaker

role = sagemaker.get_execution_role()

hub = {
    'HF_MODEL_ID': 'cross-encoder/nli-deberta-v3-large',
    'HF_TASK': 'text-classification'
}

huggingface_model = HuggingFaceModel(
    env=hub,
    role=role,
    transformers_version='4.37',
    pytorch_version='2.1',
    py_version='py310'
)

predictor = huggingface_model.deploy(
    initial_instance_count=1,
    instance_type='ml.g5.xlarge',
    endpoint_name='nli-deberta-v3-large'
)

print(f"NLI endpoint deployed: {predictor.endpoint_name}")
```

Run deployment:
```bash
python deploy_nli.py
```

### Step 3: Deploy Fast & Faithful Encoder

Deploy the Fast & Faithful encoder model:

```python
# deploy_encoder.py
from sagemaker.huggingface import HuggingFaceModel
import sagemaker

role = sagemaker.get_execution_role()

# Fast & Faithful model from HuggingFace
hub = {
    'HF_MODEL_ID': 'llm-semantic-router/fast-faithful-32k',  # Example - check actual model ID
    'HF_TASK': 'token-classification'
}

huggingface_model = HuggingFaceModel(
    env=hub,
    role=role,
    transformers_version='4.37',
    pytorch_version='2.1',
    py_version='py310'
)

predictor = huggingface_model.deploy(
    initial_instance_count=1,
    instance_type='ml.g5.2xlarge',  # Need more VRAM for 32K context
    endpoint_name='fast-faithful-verifier'
)

print(f"Encoder endpoint deployed: {predictor.endpoint_name}")
```

Run deployment:
```bash
python deploy_encoder.py
```

**Note:** If Fast & Faithful model is not available, you can use LettuceDetect as a fallback:
```python
hub = {
    'HF_MODEL_ID': 'vectara/hallucination_evaluation_model',
    'HF_TASK': 'text-classification'
}
```

### Step 4: Configure Environment

Create `.env` from template:

```bash
cp .env.example .env
```

Edit `.env`:

```bash
# AWS Configuration
AWS_REGION=us-east-1
AWS_PROFILE=default

# Bedrock Configuration
BEDROCK_KB_ID=YOUR_KNOWLEDGE_BASE_ID
BEDROCK_MODEL_ID=anthropic.claude-sonnet-4-20250514

# SageMaker Endpoints
SAGEMAKER_NLI_ENDPOINT=nli-deberta-v3-large
SAGEMAKER_ENCODER_ENDPOINT=fast-faithful-verifier

# Set to false for real AWS mode
MOCK_MODE=false
```

### Step 5: Run Comparison

```bash
# Run full comparison on AWS
python compare.py

# Generate report
python report.py
```

Results will be saved to:
- `data/comparison_results.json` - Detailed per-case results
- `data/aggregate_metrics.json` - Summary statistics
- `COMPARISON_REPORT.md` - Formatted markdown report

## Usage Examples

### Individual Evaluators

Test each evaluator independently:

```python
from evaluators import LLMJudgeEvaluator, NLIClaimsEvaluator, RealtimeEncoderEvaluator

# LLM-as-Judge
llm_judge = LLMJudgeEvaluator(mock_mode=False)
result = llm_judge.evaluate(context, query, response)
print(f"Faithfulness: {result.faithfulness_score:.3f}")
print(f"Latency: {result.latency_ms:.2f}ms")
print(f"Cost: ${result.estimated_cost_usd:.6f}")

# NLI Claims
nli_claims = NLIClaimsEvaluator(mock_mode=False)
result = nli_claims.evaluate(context, query, response)
for claim in result.claims:
    print(f"Claim: {claim.claim}")
    print(f"  Entailment: {claim.entailment_score:.3f}")

# Real-Time Encoder
encoder = RealtimeEncoderEvaluator(mock_mode=False)
result = encoder.evaluate(context, query, response)
if result.hallucinated_spans:
    print("Hallucinated spans:")
    for span in result.hallucinated_spans:
        print(f"  - {span.text}")
```

### RAG Pipeline

Run the full RAG pipeline:

```python
from rag_pipeline import BedrockRAGPipeline

pipeline = BedrockRAGPipeline(mock_mode=False)
result = pipeline.run("What is Amazon S3?")

print(f"Query: {result.query}")
print(f"Response: {result.generated_response}")
print(f"Latency: {result.total_latency_ms:.2f}ms")
```

## Cost Estimation

Based on 1 million evaluations per month:

| Approach | Per Eval | Monthly Cost (1M) | Yearly Cost |
|----------|----------|-------------------|-------------|
| LLM-as-Judge | $0.0058 | $5,800 | $69,600 |
| NLI Claims | $0.000073 | $73 | $876 |
| Real-Time Encoder | $0.000011 | $11 | $132 |

**SageMaker Instance Costs:**
- ml.g5.xlarge (NLI): ~$1.41/hour (~$1,000/month if always on)
- ml.g5.2xlarge (Encoder): ~$2.03/hour (~$1,500/month if always on)

**Tip:** Use SageMaker Serverless Inference for bursty workloads to reduce costs.

## Performance Benchmarks

From testing on 30 test cases (results may vary based on context length and complexity):

| Metric | LLM-as-Judge | NLI Claims | Real-Time Encoder |
|--------|--------------|------------|-------------------|

## Troubleshooting

### Mock Mode Not Working

If mock mode fails:
```bash
# Ensure MOCK_MODE is set
export MOCK_MODE=true
python compare.py
```

### SageMaker Endpoint Errors

If SageMaker endpoints fail:
```bash
# Check endpoint status
aws sagemaker describe-endpoint --endpoint-name nli-deberta-v3-large

# Check CloudWatch logs
aws logs tail /aws/sagemaker/Endpoints/nli-deberta-v3-large
```

### Bedrock Access Denied

Ensure your IAM role has Bedrock permissions:
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "bedrock:InvokeModel",
        "bedrock:Retrieve"
      ],
      "Resource": "*"
    }
  ]
}
```

### Out of Memory (OOM) on SageMaker

If encoder model OOMs:
- Increase instance size: ml.g5.2xlarge → ml.g5.4xlarge
- Reduce batch size in inference code
- Truncate long contexts to 16K tokens

## Development

### Adding New Evaluators

To add a new verification approach:

1. Create `evaluators/your_evaluator.py`
2. Inherit from a base pattern (see existing evaluators)
3. Implement `evaluate(context, query, response)` method
4. Add to `evaluators/__init__.py`
5. Update `compare.py` to include your evaluator
